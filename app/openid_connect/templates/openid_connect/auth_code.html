Authorization Code Flow

The authorization code flow is used to obtain both access and refresh tokens by exchanging a code. Because it's a redirection-based flow, the client must be able to interact with the resource owner's user agent (typically a web browser), and capable of receiving incoming requests from the authorization server.

Step One: Authorization Endpoint

The first step of the flow is to redirect to the authorization endpoint. This URL includes the client ID, requested scopes, and redirect URI. Once successful, the authorization server will return a code.
