Implicit Grant Flow

The implicit grant flow is used for single-page applications (like JavaScript or React.js) to obtain access and id tokens without having to exchange a code. Although this is rarely used due to its lack of security, it offers the ability to see the tokens within the URL.

Step One: Authorization

The first step of the flow is to redirect to the authorization endpoint. This URL includes the client ID, requested scopes, and redirect URI. Note that with this flow, we are setting the grant type to "token," which will return the tokens in the callback URL, as opposed to a code.
