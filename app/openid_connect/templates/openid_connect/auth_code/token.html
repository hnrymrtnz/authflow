Authorization Code Flow

The authorization code flow is used to obtain both access and refresh tokens by exchanging a code. Because it's a redirection-based flow, the client must be able to interact with the resource owner's user agent (typically a web browser), and capable of receiving incoming requests from the authorization server.

Step Three: Token Decode

After exchanging the code for the token(s), you'll be able to decode them, and see any returned claims within the payload.

Access Token: {{ access_token }}

Id Token: {{ id_token }}

These claims are either a part of the requested scopes, or custom claims that are configured within the client.
